## Context

**Notification permission states:**
- `"default"` — User hasn't decided, show banner with request button
- `"granted"` — User allowed, banner hides, notifications work
- `"denied"` — User rejected, banner hides, don't ask again

**Browser behavior:**
- Permission dialog only appears once per browser
- User can revoke in browser settings (changes to "denied")
- iOS PWA follows iOS system notification settings

## Tasks

### Task 1: Create NotificationPermissionBanner Component

**Files:** `components/NotificationPermissionBanner.tsx`

**Action:**
Create a contextual banner component that shows/hides based on permission state:

**Component logic:**
1. On mount: check if Notification API exists and permission is "default" (undecided)
2. If permission is "default": set `showBanner = true`
3. If permission is "granted" or "denied": set `showBanner = false`
4. Store in React state

**Display when undecided:**
```
┌────────────────────────────────────────────────┐
│ Alışkanlık bildirimlerini almak için izin verin │
│ [İzin Ver]                                      │
└────────────────────────────────────────────────┘
```

**Styling:**
- Container: `bg-medium/20 border border-medium p-4 rounded-lg mb-4`
- Text: `text-sm text-fg`
- Button: `px-4 py-2 bg-medium text-white rounded-lg text-sm`
- Button hover: standard opacity-90

**On click:**
1. Call `requestNotificationPermission()`
2. If granted: hide banner by setting `showBanner = false`
3. If denied or error: leave banner visible (or show error toast, optional)

**Null check:**
```typescript
if (!showBanner) return null;
```

**Edge cases:**
- Notification API not supported: banner never shows
- Permission already decided: banner never shows
- User clicks multiple times: debounce or disable button during request

Full example structure:
```typescript
"use client";

import { useState, useEffect } from "react";
import { requestNotificationPermission } from "@/lib/notification-permissions";

export function NotificationPermissionBanner() {
  const [showBanner, setShowBanner] = useState(false);
  const [requesting, setRequesting] = useState(false);

  useEffect(() => {
    if (typeof window === "undefined") return;
    if (!("Notification" in window)) return;
    
    if (Notification.permission === "default") {
      setShowBanner(true);
    }
  }, []);

  const handleRequestPermission = async () => {
    setRequesting(true);
    try {
      const granted = await requestNotificationPermission();
      if (granted) {
        setShowBanner(false);
      }
    } catch (err) {
      console.error("Permission request failed:", err);
    } finally {
      setRequesting(false);
    }
  };

  if (!showBanner) return null;

  return (
    <div className="bg-medium/20 border border-medium p-4 rounded-lg mb-4">
      <p className="text-sm text-fg mb-2">
        Alışkanlık bildirimlerini almak için izin verin
      </p>
      <button
        onClick={handleRequestPermission}
        disabled={requesting}
        className="px-4 py-2 bg-medium text-white rounded-lg text-sm hover:opacity-90 disabled:opacity-50"
      >
        {requesting ? "Yükleniyor..." : "İzin Ver"}
      </button>
    </div>
  );
}
```

**Verify:**
- Component renders without errors
- Banner shows when permission is "default"
- Banner hides after permission granted
- Button click calls `requestNotificationPermission()`
- Button is disabled during request

**Done:**
- `components/NotificationPermissionBanner.tsx` created and exported
- Banner shows/hides based on permission state
- Permission request works on click

## Notes

**Permission state management:**
- `Notification.permission` is read-only, set by browser
- We only read it, never set it manually
- Always check `typeof window !== "undefined"` before accessing `Notification`
- Permission check is synchronous (`Notification.permission` is a string), no async needed

**Banner placement:**
- Placed early in settings (after Kimlik) because permissions are setup-related
- Before Alışkanlıklar (which are user content, not setup)
- After Identity because settings → notifications is logical flow

**Disclaimer text styling:**
- Should be visible but not prominent (muted color)
- `text-fg/60` = 60% opacity of foreground color
- Use same font as other body text (no special styling)
- Keep it short: 2-3 lines max

**iOS-specific behavior:**
- Permission dialog appears as iOS system alert (native behavior)
- Users expect this dialog; it's familiar from other apps
- On iOS PWA, notification permission is separate from web browser
- Test thoroughly on real iOS device; simulator behavior may differ

**Future enhancements (out of scope):**
- [ ] "Test notification" button to fire sample notification immediately
- [ ] Notification time display for each habit in settings
- [ ] Notification mute option (temp disable all notifications)
- [ ] Notification history/log (showing when last notification fired)
- These are Phase 6+ features
