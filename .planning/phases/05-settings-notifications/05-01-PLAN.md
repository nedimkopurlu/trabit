**Output:**

## Tasks

### Task 1: Create ThemeToggle Component

**Files:** `components/ThemeToggle.tsx`

**Action:**
Create a toggle button component that:
1. On mount, reads current theme from localStorage via `getInitialTheme()`
2. Stores theme state in React (`useState`)
3. On click, switches to opposite theme, calls `applyTheme()`, updates React state
4. Displays current mode: "🌙 Gece Modu" (dark mode) or "☀️ Gündüz Modu" (light mode)
5. Button styling: `px-4 py-2 rounded-lg bg-surface text-fg hover:opacity-80`

Use `useEffect` to hydrate initial theme from localStorage on component mount (prevents mismatch with server-side theme).

**Verify:**
- Component renders without errors
- Clicking toggle switches theme immediately
- `document.documentElement.classList` contains/removes "dark" class

**Done:**
- ThemeToggle.tsx created and exports default component
- Clicking changes HTML class
- Component accepts no props (self-contained)

### Task 2: Create IdentitySentenceForm Component

**Files:** `components/IdentitySentenceForm.tsx`

**Action:**
Create an identity sentence edit form component that:
1. Display current sentence from AuthContext (or "Henüz ayarlanmamış" if empty)
2. On "Düzenle" click, switch to edit mode
3. In edit mode: show input field (maxLength 100), Submit and Cancel buttons
4. On Submit:
   - Trim whitespace
   - Validate not empty → error toast if invalid
   - Call `updateIdentitySentence(user.uid, trimmed)`
   - Update AuthContext via `setIdentitySentence(trimmed)`
   - Show success toast: "✓ Kimlik cümlesi güncellendi"
   - Exit edit mode
5. On Cancel: exit edit mode, reset input to current value
6. Disable buttons during submission

Error handling:
- Show error toast if Firestore update fails: "Güncelleme başarısız oldu"
- Log error to console

**Verify:**
- Form displays current identity sentence
- Edit mode shows input field
- Submit works, updates Firestore, shows success toast
- Cancel exits without saving
- Validation prevents empty submission

**Done:**
- IdentitySentenceForm.tsx created
- Form submits valid input to Firestore
- User sees success/error feedback via toasts
- AuthContext updates immediately after Firestore write

## Notes

**Theme implementation details:**
- `getInitialTheme()` reads from localStorage, falls back to system preference (via `window.matchMedia`)
- `applyTheme()` adds/removes `dark` class on `<html>` element and writes to localStorage
- Tailwind dark mode class is pre-configured (`darkMode: "class"` in tailwind.config.ts)
- CSS variables in globals.css already handle light/dark color mappings
- `ThemeScript.tsx` runs in `<head>` to prevent flash on page load (already in place)

**Identity sentence details:**
- Max 100 characters enforced by `<input maxLength={100}>`
- Trimmed on submit to remove accidental spaces
- Empty check validates string is not empty or whitespace-only
- AuthContext updates immediately after Firestore write, no need to refetch
- Toast feedback uses existing ToastContext pattern

**Common pitfalls to avoid:**
- Don't call `applyTheme()` before checking localStorage is available (already handled in lib/theme.ts via `typeof window !== "undefined"`)
- Don't forget to clear input on Cancel — reset to current `identitySentence` from AuthContext
- Don't disable form indefinitely — always clear `submitting` state in finally block
- Test on actual iOS PWA device, not just browser—localStorage behavior differs
