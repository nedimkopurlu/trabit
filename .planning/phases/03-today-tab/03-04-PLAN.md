**Action:**
Create full-screen modal that appears when all habits are completed.

**Interface:**
```typescript
interface CelebrationScreenProps {

  identitySentence: string; // User's affirmation (e.g., "Gücendim!")
  onDismiss: () => void; // Callback to close
}
```

**Implementation:**
1. Full-screen overlay with Framer Motion animations:
   - Background: `fixed inset-0 z-50 bg-black/50 backdrop-blur-sm`
   - Modal: `flex items-center justify-center`
2. Modal content (centered card):
   - `bg-white dark:bg-neutral-900`
   - `rounded-2xl p-8`
   - `max-w-md mx-4`
3. Animations (Framer Motion):
   - `initial: { scale: 0.5, opacity: 0 }`
   - `animate: { scale: 1, opacity: 1, transition: { type: "spring", stiffness: 100, damping: 20 } }`
   - `exit: { scale: 0.5, opacity: 0 }`
4. Content layout:
   ```
   🎉 (emoji burst, text-6xl)
   
   "Tamamladın!"
   (text-3xl font-bold)

   [Dismiss button]
   ```
5. Dismiss button:
   - `bg-green-500 text-white px-6 py-2 rounded-lg`
   - `onClick={onDismiss}`
   - Text: "Devam et" (Continue)
6. Auto-dismiss after 6 seconds (allow user to see, then fade)

### Task 2: Integrate Celebration into Today Page
**Files:** `app/today/page.tsx` (MODIFY)

**Action:**
Add celebration logic to Today page to track completion count and show modal when all habits done.

**Implementation:**
1. Track completion count:
   ```typescript
   const completionStates = todayHabits.map(h => useHabitCompletion(h.id));
   const completedCount = completionStates.filter(c => c.isComplete).length;
   ```
   (Note: This requires calling hook for each habit, which happens in TodayHabitCard. Alternative: track in Firestore query for efficiency, deferred to Phase 3.1)

2. Show celebration when:
   ```typescript
   const showCelebration = todayHabits.length > 0 && completedCount === todayHabits.length;
   ```

3. Render with `AnimatePresence`:
   ```typescript
   <AnimatePresence>
     {showCelebration && (
       <CelebrationScreen
         habitCount={completedCount}
         identitySentence={identitySentence}
         onDismiss={() => setShowCelebration(false)}
       />
     )}
   </AnimatePresence>
   ```

4. State management:
   - `const [showCelebration, setShowCelebration] = useState(false);`
   - Auto-dismiss updates state via modal's auto-dismiss

**Alternative approach (more efficient for Phase 3.1):**
Query Firestore for completion count directly:
```typescript
const [completedCount, setCompletedCount] = useState(0);
useEffect(() => {
  const query = query(collection(db, `users/${uid}/habits`), 
    where("completedToday", "==", true));
  // Listen to query changes
}, [uid]);
```
Requires schema update to track `completedToday` flag. Deferred to Phase 3.1.

## Notes

### Why Optional?
Celebration is "nice to have" for Phase 3. Core requirement is filtering, sorting, completing, and animating. Celebration is the polish/motivation layer. If time is tight in execution:
1. Complete Plans 03-01, 03-02, 03-03 first (mandatory)
2. If time remains, add Plan 03-04
3. Otherwise, schedule as Phase 3.1

### Auto-Dismiss Timing
6 seconds chosen to:
- Give user time to feel the achievement (~2s read + 4s for momentum)
- Return to habit list before scrolling gets confused
- Not too long (feel like getting stuck)
- Customizable per user in Phase 4 (settings)

### Phase 3.1 Roadmap
Plan 03-04 sets up structure for Phase 3.1 enhancements:
- **Confetti animation:** Wire `useCelebrationConfetti()` to react-confetti or canvas
- **Celebration sound:** Wire `playCelebrationSound()` to audio file
- **Emoji animations:** Enhance celebration modal with shaking, bouncing emoji
- **Streak tracking:** Show "3-day streak! 🔥" in celebration
- **Analytics:** Track completion patterns, celebrate milestones

### Firestore Optimization (Phase 3.1)
Current approach calls `useHabitCompletion()` for each habit to track completion count. This works but could be optimized:
- Future: Query `users/{uid}/completions/{todayDate}` directly
- Requires schema change (flatten completions for easier querying)
- Benefit: Reduce hook calls, faster rendering
- Deferred to Phase 3.1 to avoid over-engineering Phase 3

### Testing Note
Celebration modal testing is tricky with auto-dismiss. Use `vi.useFakeTimers()` in tests to advance time without actual waiting.
