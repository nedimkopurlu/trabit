### Task 1: Create Toast Context & Provider
**Files:** `lib/toast-context.tsx`, `lib/toast-types.ts`

**Action:**
Create React Context for app-wide toast notifications (custom, lightweight alternative to react-toastify).

**Interfaces:**
```typescript
// lib/toast-types.ts
export interface Toast {
  id: string; // Unique ID
  message: string;
  type: "success" | "error" | "info"; // Color mapping
  duration?: number; // ms before auto-dismiss (default 3000)
  icon?: string; // Optional emoji or icon
}

// lib/toast-context.tsx
interface ToastContextType {
  toasts: Toast[];
  addToast: (message: string, type: Toast["type"], duration?: number) => void;
  removeToast: (id: string) => void;
}

export const ToastContext = createContext<ToastContextType | undefined>(undefined);
export const useToast = (): ToastContextType => { /* throw if not in provider */ }
export const ToastProvider: React.FC<{ children: React.ReactNode }> => { /* manage state */ }
```

**Implementation:**
1. `ToastProvider` component manages `toasts` state (array)
2. `addToast(message, type, duration)`:
   - Generate unique ID (e.g., `${Date.now()}-${Math.random()}`)
   - Add to array
   - Set timeout to auto-remove after duration (default 3000ms)
3. `removeToast(id)`: Filter out by ID
4. Render `ToastContainer` child component (created in Task 2)
5. Export `useToast()` hook for consuming components

**Duration defaults:**
- Success: 3000ms (quick, positive)
- Error: 5000ms (longer, needs reading)
- Info: 4000ms (neutral)

**Verify:**

### Task 3: Create Today Page with Habit Filtering
**Files:** `app/today/page.tsx`, `app/today/layout.tsx` (if needed)

**Action:**
Create Today tab page that filters habits by schedule and renders habit cards.

**Implementation:**
1. Server component structure:
   ```typescript
   export default function TodayPage() {
     return <TodayPageClient />;
   }
   
   "use client"
   function TodayPageClient() { /* client logic */ }
   ```
   (Reason: separates server-side from client to enable proper data fetching)

2. `TodayPageClient` component:
   - Use `useHabits()` hook from Phase 2 to fetch habits
   - Use `useAuth()` to get `uid` and identity sentence
   - Use `useEffect` + `useMemo` to filter habits:
     ```typescript
     const timezone = getUserTimezone();
     const todayHabits = useMemo(() => {
       return habits
         .filter(h => isHabitScheduledToday(h.schedule, timezone))
         .sort((a, b) => {
           // Sort by importance: critical (0) > medium (1) > low (2)
           const importanceOrder = { critical: 0, medium: 1, low: 2 };
           return importanceOrder[a.importance] - importanceOrder[b.importance];
         });
     }, [habits, timezone]);
     ```

3. Render structure:
   ```
   <main className="p-4 pb-20">
     <h1>Bugün</h1>
     <AnimatePresence mode="popLayout">
       {todayHabits.map(habit => (
         habit.type === "boolean" 
           ? <TodayHabitCard key={habit.id} habit={habit} />
           : <AmountHabitCard key={habit.id} habit={habit} />
       ))}
     </AnimatePresence>
   </main>
   ```

4. Empty state:
   - If `todayHabits.length === 0`: Show "Bugün alışkanlık yok" message

5. Loading state:
   - If `loading === true` (from useHabits hook): Show loading spinner

6. Wrap in `ToastProvider` (or ensure app.tsx wraps entire app in ToastProvider)

**Integration with toast:**
- When habit completes (from HabitCheckButton onClick), trigger toast
- Access toast via `useToast()` in TodayHabitCard
- Message format: `✓ {habit.name} tamamlandı! {identitySentence}`
- Example: `✓ Yoga tamamlandı! Gücendim!` (gücendim = I'm proud, from auth identity sentence)

**PWA considerations:**
- Page works offline (habits cached, completions sync when online)
- Responsive mobile-first layout: full width on mobile, card width on tablet
- Safe area padding on iPhone: `pb-20` prevents content hiding under home bar

## Notes

### Toast Container Positioning
Fixed position at `bottom-4 right-4` on desktop, adapts to mobile viewport safe area. On iPhone, respects notch and home indicator.

### Identity Sentence Integration
Identity sentence comes from Phase 1 auth context (e.g., user's affirmation like "Gücendim!" or "Ben başarıyım!"). Passed to toast message for personalization and motivation.

### Why Custom Toast?
- ✓ Lightweight: ~2KB vs 60KB for react-toastify
- ✓ Framer Motion animations: consistent with habit card animations
- ✓ No external dependencies: one less thing to track
- ✗ Less feature-rich: no queues, no custom callbacks, but fine for PWA

### Offline Behavior
When offline:
- Today page loads from local cache (next-pwa handles)
- Habit completions marked optimistically
- Toast shows immediately (no wait for network)
- On reconnection, Firestore listener syncs (no user action needed)

### Color Coding (TODAY-02)
Habit importance already defined in Phase 2 schema with `color` field. Cards inherit this color for progress bar and visual prominence. Border colors automatically adjust based on completion state.

### Celebration Screen (Deferred)
When all habits complete for today, `todayHabits.length > 0 && completedCount === todayHabits.length` triggers celebration. Deferred to Phase 3.1. For now, last habit completion just shows success toast.
