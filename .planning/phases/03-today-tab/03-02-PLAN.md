### Task 1: Create ProgressBar Component
**Files:** `components/ProgressBar.tsx`, `components/ProgressBar.test.tsx`

**Action:**
Create reusable progress bar component using Framer Motion spring animation.

**Interface:**
```typescript
interface ProgressBarProps {
  current: number; // Current value (0+)
  target: number; // Target value (1+)
  color: string; // Hex color (e.g., "#ef4444" for red)
  size?: "sm" | "md" | "lg"; // Height: sm=h-2, md=h-3, lg=h-4
  animated?: boolean; // Start from 0 if true, else jump to current
}
```

**Implementation:**
1. Calculate percentage: `Math.min((current / target) * 100, 100)`
2. Render outer container: `rounded-full bg-gray-200 dark:bg-neutral-700 overflow-hidden`
3. Render inner bar with Framer Motion:
   - `initial: { width: animated ? 0 : "${percentage}%" }`
   - `animate: { width: "${percentage}%" }`
   - `transition: { duration: 0.8, type: "spring", stiffness: 100, damping: 30 }`
4. Bar color from props, respects dark mode
5. Clamp to 100% visually even if current > target (mathematically)

**Test cases:**
- Progress bar fills to 50% when current=5, target=10
- Progress bar animates from 0% to final % when `animated={true}`
- Progress bar jumps to final % immediately when `animated={false}`
- Different sizes (sm, md, lg) render different heights
- Color prop applies to bar, respects hex values

**Verify:**
```bash

```

**Done:**
- Component exported, accepts required props
- Animation uses Framer Motion spring (not linear)
- Tests passing
- No TypeScript errors

### Animation Philosophy
Framer Motion spring physics creates more natural feel than linear easing:
- `stiffness: 100, damping: 30` = bouncy but controlled
- `duration: 0.8` on progress bar = feels organic, not rushed
- Exit animations on individual cards trigger automatically in AnimatePresence (handled in Plan 03-03)

### Why "2dk" Button?
The "2dk" (2 minute quick) button is a separate UI element that may trigger a different completion type in future. For Phase 3, it can call the same `markComplete()` but with `quick: true` flag. Implementation deferred to Task 1 in Plan 03-03.

### Handling Offline
Framer Motion animations work offline. `useHabitCompletion` hook handles Firestore sync failures gracefully (optimistic updates still show locally while pending sync).

Like Plan 03-01, this creates test files with failing tests. Components must be implemented to pass tests before moving to Wave 1 integration.
