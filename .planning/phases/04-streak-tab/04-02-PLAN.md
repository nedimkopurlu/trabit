Create a reusable 7-day heat map grid component that displays color-coded completion status cells.

**Component API:**
```typescript
interface HeatMapGridProps {
  cells: HeatMapCell[];          // Array of 7 HeatMapCell objects from getLast7DayWindow
  showTooltip?: boolean;         // Default: true; show date tooltip on hover
  animated?: boolean;            // Default: true; stagger animation on mount
}

export default function HeatMapGrid(props: HeatMapGridProps) { ... }
```

**Implementation approach:**

1. **Grid Layout:**
   - Use Tailwind: `grid grid-cols-7 gap-2`
   - Each cell: responsive size, square aspect ratio
   - On mobile: full width ÷ 7 = each cell ~12% of width
   - On desktop: fixed cell size ~48px

2. **Cell Rendering:**
   - For each HeatMapCell, render a div with:
     * Tailwind class from `STATUS_COLORS[cell.status]`
     * Square aspect ratio: `aspect-square` with `rounded`
     * Border for missed days: `border-2 border-gray-200`
     * Hover state: `hover:scale-110` (slight zoom on hover)
   - Optional tooltip on hover: Date (e.g., "Mon, Apr 8") or just day-of-week

3. **Framer Motion Animation:**
   - On mount, stagger cells in from left to right with opacity fade
   - Use `motion.div` for each cell
   - Stagger: `delayChildren: 0.05, staggerChildren: 0.1`
   - Each cell: `initial: { opacity: 0, scale: 0.8 }`, `animate: { opacity: 1, scale: 1 }`
   - Duration: 0.3s per cell
   - Reference: Phase 3 TodayHabitCard exit animation pattern

4. **Dark Mode:**
   - Missed cell border: `dark:border-gray-600`
   - Irrelevant (gray): `dark:bg-gray-700`
   - Test in both light and dark modes

5. **Accessibility:**
   - Add ARIA labels: `aria-label={`${cell.dayOfWeek} - ${cell.status}`}`
   - Semantic structure: `<div role="grid">`

**Implementation details:**
- Import HeatMapCell type and STATUS_COLORS from heat-map-utils.ts
- No external libraries except Framer Motion
- Component is presentational: receives data, doesn't fetch
- Tooltip text can be simple: just the dayOfWeek + status

  <name>Task 2: StatRow Component (components/StatRow.tsx)</name>
  <files>components/StatRow.tsx</files>
  <action>
Create a component to display a single statistic (number + label) with animated number counter.

**Component API:**
```typescript
interface StatRowProps {
  value: number;                 // The statistic value (e.g., 5 for 5-day streak)
  label: string;                 // Label text (e.g., "Current Streak")
  suffix?: string;               // Optional suffix (e.g., "%" for percentage, "🔥" for streaks)
  icon?: React.ReactNode;        // Optional icon component
  variant?: "default" | "highlight"; // Highlight: larger text, more emphasis
}

export default function StatRow(props: StatRowProps) { ... }
```

**Implementation approach:**

1. **Layout:**
   - Flex row: `flex items-center justify-between` or `flex flex-col items-start gap-2`
   - Number on left/top, label on right/bottom
   - Variants:
     * default: normal-sized number, small gray label
     * highlight: larger number (text-2xl), bolder label

2. **Animated Number Counter:**
   - Use Framer Motion `motion.div` for the number
   - When value changes (new prop), animate from old → new value
   - Animation: `initial: { opacity: 0.5, y: 10 }`, `animate: { opacity: 1, y: 0 }`
   - Duration: 0.4s, easing: easeInOut
   - Reference: Phase 3 StatRow pattern (if used; or similar number animations in Framer Motion docs)

3. **Styling:**
   - Number: `text-xl md:text-2xl font-bold` (or variant adjusts size)
   - Label: `text-sm opacity-60 dark:opacity-70`
   - Full row padding: `p-3` or `p-4`
   - Subtle background: `bg-gray-50 dark:bg-gray-900 rounded-lg`

4. **Optional Icon:**
   - If icon provided, render before number
   - Icon size: ~16px for labels, ~20px for highlights
   - Color: match text color

5. **Number Formatting:**
   - If suffix is "%", format as `${value}%`
   - Otherwise just: `${value}${suffix || ''}`

**Implementation details:**
- No TypeScript utility for animated number display needed (simple Framer Motion layout)
- Component is stateless: receives value prop, animates changes
- Re-renders when parent updates (e.g., useStreakStats updates)

## Notes

### Design Philosophy
- **Reusable components:** HeatMapGrid and StatRow can be used in other contexts
- **Separated concerns:** Components focus on display; utilities handle logic
- **Animation performance:** Stagger animations use GPU acceleration (transform, opacity)
- **Mobile-first:** Grid responsive without media queries via Tailwind

### Styling Details
- Heat map cells: 48px fixed on desktop, responsive on mobile
- Card shadows: `shadow-sm` for subtle depth
- Colors: match existing color palette from Phase 3 (critical=red, medium=yellow, low=green)
- Dark mode: use `dark:` prefix for all color/bg values
