<task type="auto">
  <name>Task 1: Streak Calculation Utilities (lib/streak-stats.ts)</name>
  <files>lib/streak-stats.ts</files>
  <action>
Create a pure JavaScript utilities module for streak calculations. This module takes an array of HabitCompletion objects and calculates all four statistics used in Phase 4.

**Functions to implement:**

1. `calculateCurrentStreak(completions, schedule, timezone)`: 
   - Traverse completions array backward (newest first)
   - Count consecutive days with `quick: false` (full completions only)
   - Stop counting at first day that:
     * Has no completion, OR
     * Habit is not scheduled that day (weekday habit on weekend = irrelevant, not a miss)
   - Return count; if no today completion, current streak = 0
   - Reference: Research 04-RESEARCH.md Pitfall 3 (irrelevant days not counted as misses)

2. `calculateLongestStreak(completions, schedule, timezone)`:
   - Same backward traversal as current, but find longest unbroken sequence across entire history
   - Track max streak seen
   - Reset counter when hitting a "real miss" (scheduled but not completed)
   - Return max value

3. `calculateTotalCompletions(completions)`:
   - Count all completions where `quick: false` (exclude quick/2dk)
   - Reference: STREAK-05 requirement

4. `calculateThisWeekPercentage(completions, schedule, timezone)`:
   - Define this week as Monday (today) going backward to previous Monday (7 calendar days)
   - Use ISO week definition (Mon-Sun)
   - Count completions in this range where `quick: false`
   - Count expected completions: iterate each day, check if habit is scheduled
   - Return percentage: (completed / expected) × 100, round to nearest integer
   - Handle edge case: if expected = 0 (habit not scheduled this week), return 0
   - Reference: STREAK-06, Pitfall 5 (week boundaries)

**Type definitions:**
Export `StreakStats` interface:
```typescript
export interface StreakStats {
  current: number;      // Today's streak count
  longest: number;      // All-time longest
  total: number;        // Total full completions
  thisWeekPercentage: number; // 0-100
}
```

**Implementation notes:**
- Use only native JavaScript (no external libs)
- All date handling via getTodayInTimezone + formatDateAsKey from habit-schedule.ts
- Completions should arrive pre-sorted by date (ascending); reverse for backward traversal
- Memoization happens in React hook (Plan 04-03), not here
- Pure functions: same input = same output always

  <name>Task 2: Heat Map Window & Color Mapping (lib/heat-map-utils.ts)</name>
  <files>lib/heat-map-utils.ts</files>
  <action>
Create utilities to generate the 7-day heat map grid and map completion statuses to colors.

**Functions to implement:**

1. `getLast7DayWindow(timezone, completionsByDate)`:
   - Generate 7 calendar days: 6 days before today + today
   - Use getTodayInTimezone to get today's date in user's timezone
   - For each day, calculate date string (YYYY-MM-DD) and day-of-week name ("Mon", "Tue", etc.)
   - Use Intl.DateTimeFormat for weekday name generation (consistent with Phase 3 patterns)
   - Return array of HeatMapCell objects sorted oldest→newest
   - Input `completionsByDate` is a Map<dateStr, HabitCompletion> for O(1) lookup
   - Leave status as "empty" initially; caller will fill with mapCompletionToStatus

2. `mapCompletionToStatus(completion, habit, schedule, timezone)`:
   - If no completion exists: check if habit scheduled that day
     * Scheduled: status = "missed" (white)
     * Not scheduled (irrelevant): status = "irrelevant" (gray)
   - If completion exists:
     * completed: true, quick: false → "completed" (green)
     * completed: true, quick: true → "quick" (yellow)
   - Return status color name for CSS class mapping
   - Reference: STREAK-01, STREAK-02, STREAK-03 requirements

**Type definitions:**
```typescript
export interface HeatMapCell {
  date: string;           // YYYY-MM-DD
  dayOfWeek: string;      // "Mon", "Tue", "Wed", etc.
  status: "completed" | "quick" | "irrelevant" | "missed";
  tooltip?: string;       // For hover: "Mon, Apr 8 - Completed"
}
```

**CSS Color Mapping (for components to use):**
```typescript
export const STATUS_COLORS: Record<HeatMapCell['status'], string> = {
  completed: "bg-green-500",      // Full completion
  quick: "bg-yellow-400",          // 2dk quick completion
  irrelevant: "bg-gray-300",       // Not scheduled (weekend habit on weekday, etc.)
  missed: "bg-white border-2 border-gray-200", // Scheduled but empty
};
```

**Implementation notes:**
- Both functions are pure: same input = same output
- Timezone-aware: getLast7DayWindow must use user's timezone for day boundaries
- Use Map for completions lookup (faster than array.find in loop)
- Component will use STATUS_COLORS to render Tailwind classes
- Tooltip dates use browser Intl API for localization (optional but nice)

## Notes

### Architecture
- **No hand-rolling:** All calculations use Firestore listeners and native JS
- **Timezone-aware:** All functions accept timezone parameter; default to user's browser timezone
- **Schedule-aware:** Irrelevant days (not scheduled) do NOT count as misses
- **Quick completion handling:** Skipped in current/longest/total, but shown separately in heat map colors

### Pitfalls Avoided (from Research 04-RESEARCH.md)
- ✅ Pitfall 1 (Timezone mismatches): Use getTodayInTimezone consistently
- ✅ Pitfall 2 (Quick breaks streak): Filter to `quick !== true` before streak walk
- ✅ Pitfall 3 (Irrelevant days counted as misses): Apply schedule filter in traversal
- ✅ Pitfall 4 (Excessive re-renders): useMemo on [completions, schedule, timezone]
- ✅ Pitfall 5 (Week boundary issues): ISO week definition (Mon-Sun) explicit
