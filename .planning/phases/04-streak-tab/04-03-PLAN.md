**Component skeleton:**
```typescript
'use client';

import { useAuth } from '@/lib/auth-context';
import { useHabits } from '@/lib/use-habits';
import StreakHabitCard from '@/components/StreakHabitCard';

export default function SeriPage() {
  const { user, uid } = useAuth();
  const { habits, isLoading, error } = useHabits(uid);

  if (!uid) {
    return <div>Not authenticated</div>;
  }

  if (isLoading) {
    return <div className="...">Loading...</div>;
  }

  if (error) {
    return <div className="...">Error: {error}</div>;
  }

  if (habits.length === 0) {
    return <div className="...">No habits yet</div>;
  }

  return (
    <main className="px-4 py-6 pb-24">
      <h1 className="text-3xl font-bold mb-2">Seri</h1>
      <p className="text-sm opacity-60 mb-6">Track your habit streaks</p>
      
      <div className="flex flex-col gap-4">
        {habits.map(habit => (
          <StreakHabitCard key={habit.id} habit={habit} uid={uid} />
        ))}
      </div>
    </main>
  );
}
```

**Implementation details:**
- Use 'use client' directive (client-side component for hooks)
- Import useAuth from auth-context.tsx (Phase 1 pattern)
- Import useHabits from use-habits.ts (Phase 2 pattern)
- StreakHabitCard handles its own stats hooks internally
- No need to manually manage completion listeners here; each StreakHabitCard uses useStreakStats
- Page is data-driven: changes to habits or completions flow through component tree

**Real-time sync:**
- When user marks habit complete (in Today tab or elsewhere):
  1. Firestore listener in useStreakStats fires
  2. Stats state updates in StreakHabitCard
  3. Card re-renders with new heat map and stats
  4. All happens automatically via React hooks

**Error Recovery:**
- If one habit card fails to load stats: show `--` for stats values
- If entire page fails: show error banner with retry option
- Errors don't crash the page: graceful degradation

## Notes

### Architecture Patterns Used
- **Real-time listeners:** Firestore onSnapshot for live stat updates
- **React hooks:** useStreakStats encapsulates listener + calculation logic
- **Memoization:** useMemo prevents unnecessary recalculations
- **Component composition:** Small, reusable components (HeatMapGrid, StatRow) vs. integrated card

### Performance Considerations
- Each StreakHabitCard has its own listener (one per habit)
- Listeners only update when completions for that habit change
- Calculations are O(n) where n = completions for one habit (small dataset)
- No N+1 queries: single listener per habit vs. 7 individual queries

### Mobile Optimization
- Heat map cells sized responsively (full width on mobile)
- Card layout stacks vertically on mobile
- Safe area padding prevents hiding behind TabBar
- Animations use GPU acceleration (transform, opacity)
